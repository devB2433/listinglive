# 需求说明书 (Requirements Specification)

**项目名称**: Vivid - 房地产视频生成服务
**版本**: 1.0
**日期**: 2026-03-05
**状态**: 已确认

---

## 1. 项目概述

### 1.1 项目目标
为房地产经纪人提供基于AI的图片转视频服务，通过专业的场景模板和运镜效果，快速生成高质量的房产展示视频。

### 1.2 目标用户
- 主要用户：房地产经纪人
- 地理位置：加拿大本地用户
- 使用场景：房产营销、社交媒体推广、客户展示

### 1.3 核心价值
- 降低视频制作门槛，无需专业视频编辑技能
- 快速生成专业级房产展示视频
- 提供预制场景模板，确保视觉效果一致性
- 支持批量处理，提高工作效率

## 2. 功能需求

### 2.1 视频生成功能

#### 2.1.1 短视频生成
**功能描述**: 单张图片生成单个视频

**输入参数**:
- 图片：1张（支持常见格式：JPG, PNG）
- 场景模板：从预制模板中选择（如"黄昏暖光"、"清晨阳光"等）
- 分辨率：720P / 1080P（固定选项）
- 宽高比：16:9横屏 / 9:16竖屏 / 1:1方形（固定选项）
- 视频时长：
  - 基础版：固定时长（如4秒）
  - Pro/Ultimate版：可自定义时长（2-10秒）
- Logo水印：可选，上传logo图片

**输出**:
- 单个视频文件（MP4格式）
- 自动添加转场效果（淡入淡出）

**处理方式**:
- 异步生成，提交后进入队列
- 生成完成后通知用户（站内通知 + 邮件）

#### 2.1.2 合并视频生成
**功能描述**: 多张图片生成多个视频片段，自动合并成一个完整视频

**输入参数**:
- 图片：2-10张
- 场景模板：
  - 基础版：所有图片使用统一模板
  - Pro/Ultimate版：每张图片可独立选择模板
- 视频时长：
  - 基础版：所有片段统一时长
  - Pro/Ultimate版：每个片段可独立设置时长
- 图片顺序：
  - 基础版：固定顺序（上传顺序）
  - Pro/Ultimate版：支持拖拽调整顺序
- 分辨率和宽高比：同短视频
- Logo水印：可选

**输出**:
- 单个合并后的视频文件
- 片段之间自动添加转场效果（淡入淡出）

**配额计算**:
- 按小视频数量计算（N张图片 = 消耗N个配额）

### 2.2 场景模板系统

#### 2.2.1 模板特性
- 预制模板，由系统管理员维护
- 模板名称对用户可见（如"黄昏暖光"、"清晨阳光"、"夜景灯光"）
- 模板对应的提示词对用户不可见，不可修改
- 针对房地产场景优化

#### 2.2.2 模板管理
- 管理员可在后台添加/编辑/删除模板
- 支持启用/禁用模板
- 支持模板排序

### 2.3 视频参数配置

#### 2.3.1 分辨率选项
- 720P（1280x720）
- 1080P（1920x1080）
- 固定选项，基础功能

#### 2.3.2 宽高比选项
- 16:9 横屏（适合YouTube、网站）
- 9:16 竖屏（适合TikTok、Instagram Stories）
- 1:1 方形（适合Instagram Feed）
- 固定选项，基础功能

#### 2.3.3 视频时长
- 范围：2-10秒
- 基础版：固定时长
- Pro/Ultimate版：可自定义每个片段的时长

### 2.4 Logo水印功能

**功能描述**: 在视频上添加用户的品牌Logo

**参数**:
- Logo图片：用户上传（支持PNG透明背景）
- 位置：右下角（固定）
- 大小：视频宽度的15%（固定）
- 透明度：70%（固定）
- 边距：20px（固定）

**使用方式**:
- 用户可选择是否添加水印
- 标准功能，所有用户可用

## 3. 用户系统

### 3.1 用户注册与登录

#### 3.1.1 注册方式（阶段1）
- **注册信息**：用户名、密码、邮箱（三者必填）。`生产环境`密码需至少 8 位，且包含大写字母、小写字母和特殊字符；`开发环境`可放宽以便测试。
- **防欺诈**：注册时系统向该邮箱发送**邮件验证码**，用户输入验证码通过后账号才创建成功，用于防止批量注册与欺诈。
- **流程**：用户填写用户名、密码、邮箱 → 请求发送验证码 → 系统发邮件 → 用户提交验证码完成注册。
- 注册赠送：5个小视频配额（一次性），赠送配额可使用全部功能（包括高级功能）。

#### 3.1.2 登录方式（阶段1）
- **用户名 + 密码** 或 **邮箱 + 密码** 登录（二选一即可）。
- 阶段2：Google SSO（后续实现）。

#### 3.1.3 测试账号（开发/测试环境）
- 系统初始提供一名为 **root** 的测试账号，用户名与密码均为 **root**，便于联调与测试。
- 生产环境可通过配置关闭或移除该账号。

### 3.2 套餐体系

#### 3.2.1 套餐类型
| 套餐 | 配额（小视频/月） | 可用功能 | 价格 |
|------|------------------|----------|------|
| 注册赠送 | 5个（一次性） | 全部功能 | 免费 |
| 基础版 | 可配置 | 仅基础功能 | 可配置 |
| Pro版 | 可配置 | 基础 + 高级功能 | 可配置 |
| Ultimate版 | 可配置 | 基础 + 高级功能 | 可配置 |

**说明**:
- Pro和Ultimate功能相同，仅配额数量不同
- 配额数量和价格由配置文件驱动，可灵活调整

#### 3.2.2 基础功能
- 短视频生成
- 合并视频生成（统一模板、统一时长、固定顺序）
- 选择预制场景模板
- 选择分辨率和宽高比（固定选项）
- 添加Logo水印（固定参数）
- 转场效果（默认）

#### 3.2.3 高级功能（仅Pro/Ultimate）
- 每张图片独立选择场景模板
- 每个视频片段独立设置时长
- 拖拽调整合并视频的图片顺序

### 3.3 配额管理

#### 3.3.1 配额计算
- 计费单位：小视频数量
- 短视频：1张图片 = 1个配额
- 合并视频：N张图片 = N个配额

#### 3.3.2 配额周期
- 套餐配额：按月重置
- 未使用配额：不累积到下月

#### 3.3.3 配额包
- 独立购买，增加额外配额
- 规格：可配置（如10个、30个、50个）
- 有效期：可配置（如30天、90天、永久）
- 功能权限：跟随用户套餐
  - 基础版用户购买配额包 → 只能使用基础功能
  - Pro/Ultimate用户购买配额包 → 可使用高级功能

#### 3.3.4 配额退还
- 视频生成失败 → 自动退还配额
- 视频生成成功 → 不退还配额

### 3.4 套餐管理

#### 3.4.1 订阅规则
- 套餐按月自动续费
- 支持升级：基础版 → Pro版 → Ultimate版
- 不支持降级：需等当前套餐到期后重新订阅
- 升级时按比例计算剩余时间，抵扣到新套餐

#### 3.4.2 取消订阅
- 用户可随时取消订阅
- 取消后当前周期仍有效，到期后不再续费

## 6. 视频存储与下载

### 6.1 存储时长
按套餐区分，由配置文件控制：
- 基础版：可配置（如7天、15天）
- Pro版：可配置（如30天、60天）
- Ultimate版：可配置（如90天、永久）

### 6.2 下载限制
- 存储期内：无限次下载
- 过期后：无法下载

### 6.3 存储空间
- 无总存储空间限制
- 仅按时间限制

### 6.4 存储实现约定
- **短期**：在服务器本地目录存储视频与上传文件；满足上述存储时长与下载限制即可。
- **扩展**：实现上采用存储抽象层，便于后续切换为对象存储（如 AWS S3）而无需变更业务逻辑。

**邮件内容**:
- 用户名（邮箱）
- 套餐类型
- 风格描述（正文）
- 联系电话
- 提交时间

### 8.4 限制
- 无申请频率限制
- 无奖励机制

### 12.1 性能要求
